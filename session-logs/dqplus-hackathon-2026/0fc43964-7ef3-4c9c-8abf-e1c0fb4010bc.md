## What I set up

**`.claude/hooks/save-session-log.py`** — reads the JSON Claude Code pipes to hooks (`transcript_path`, `session_id`), and copies the live transcript to `session-logs/<session_id>.jsonl`. Failures are swallowed so logging can never block a session.

**`.claude/settings.json`** — registers the script on two events:
- `Stop` — fires after every response, so the log stays current mid-session (survives crashes).
- `SessionEnd` — captures clean exits (`/clear`, quit, logout).

Both `.claude/settings.json` and `.claude/hooks/` are committed to the repo, so **anyone on the team** who runs Claude Code here gets the same behavior automatically.

## How it behaves

- Every session lands at `session-logs/<session_id>.jsonl` — the complete, faithful transcript (user/assistant messages, tool calls, results).
- Same session ID overwrites in place, so you get one up-to-date file per session, not duplicates.
- `session-logs/` is **not** gitignored, so the logs are tracked and will be committed with the repo.

The hook takes effect for **new sessions** — Claude Code loads `settings.json` at session start, so this current session won't trigger it, but the next one will.
