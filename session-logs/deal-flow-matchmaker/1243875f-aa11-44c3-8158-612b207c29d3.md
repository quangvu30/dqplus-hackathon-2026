1. **`83a538e`** — the core fix. The `api` service built from `./api`, so `COPY spine/dashboard/schemas` (repo-root dirs) had no valid build context. `schemas/` is loaded at **import time** by `spine.outreach`/`spine.sagas`, so the container crashed on startup and the API was never usable.
   - `docker-compose.yml`: `build: { context: ., dockerfile: api/Dockerfile }`
   - `api/Dockerfile`: `COPY schemas ./schemas`

2. **`38c28fa`** — port mapping. A remote `"Update port"` commit had set `8443:8443`, but uvicorn listens on `8000`. Now `8443:8000` (`compose config` confirms host `8443` → container `8000`).

Verified: the built image imports `api.app` and loads all 5 schemas (`draft`, `rank`, `verify`, `enrich`, `extract`) — the exact thing that was crashing before.

To run it: `docker compose up -d` — the API will be reachable on host port **8443**. Note that only `schemas/` is baked in; `agents/specs.yaml` and `seed/*.json` aren't needed by the API service (the supervisor reads `agents/` on the host, and seed data lives in Postgres).
